## 一、Week 1 教學定位

第一週不安排正式地震學活動，也不使用 Human Wave Demo。  
本週重點是建立後續 5 週課程會用到的共同工作環境，讓每位學員都能進入「可動手做、可繳交、可累積作品」的狀態。

本週核心目標：

> 讓每位教師完成自己的數位學習工作區，能使用 Google Drive、Google Form、Google Sheets、Google Apps Script、GitHub，並理解後續會用到的 Hugging Face、Vercel、opencode、Antigravity CLI、Hermes agent 等工具。

---

1. 說明本課程 6 週的學習路線。

## 00:00–00:15 開場與課程定位

### 教師說明重點

可以用以下說法開場：

> 這門課不是單純講地震學，也不是單純教 AI 工具。

2. 每週都有作業，但作業是累積期末作品的一部分。
3. 第一週的重點是把環境建好，後面才能順利做地震資料與教材。

### 板書或投影片文字

```text
本課程關鍵字：
地震學 × 地球物理 × 真實資料 × AI Agent × 教學設計 × 作品發表
```

## 00:15–00:35 6 週課程地圖

### 教師說明

說明 6 週課程如何逐步累積：

| 週次 | 學習重點 | 作品累積 |
|---|---|---|
| Week 1 | 建立工具與工作區 | 個人數位學習工作區 |
| Week 2 | 地震動與震度 | 震動觀測活動 |
| Week 3 | 震央定位 | 震央定位教學範例 |

## 01:40–02:10 Google Apps Script 測試

### 教學目標

讓學員完成第一個 GAS 程式，知道 GAS 可以和 Google Sheets 互動。

### 操作步驟

1. 開啟剛剛連接的 Google Sheets。
2. 點選：

```text
擴充功能 → Apps Script
```

3. 貼上以下程式：

```javascript
function courseHello() {
  SpreadsheetApp.getActiveSpreadsheet().toast("北市大暑期地球科學課程：GAS 測試成功！");
}
```

4. 儲存專案。
5. 執行 `courseHello()`。
6. 第一次執行需要授權。
7. 回到 Google Sheets，確認畫面中出現提示訊息。
8. 截圖保存到 `week01_setup` 資料夾。

### 教師說明

> 我們今天不要求大家會寫程式。  
> 今天只要知道一件事：Google Apps Script 可以幫我們把表單、試算表、文件和教學流程串起來。  
> 未來可以用它做自動整理、自動寄信、自動產生報告，甚至做一個簡單的地震資料展示系統。

### 常見問題

#### 問題 1：找不到 Apps Script

可能原因：

- Google 介面語言不同。
- 學校帳號限制。
- 使用手機或平板。

處理方式：

- 改用個人 Google 帳號。
- 使用筆電瀏覽器。
- 由教師提供備用表單。

#### 問題 2：授權時出現警告

說明：

- 這是 Google 對未驗證腳本的提醒。
- 課堂測試腳本只操作自己的試算表。
- 可點選進階後繼續。

#### 問題 3：執行後沒有反應

檢查：

- 是否在 Google Sheets 綁定的 Apps Script 中執行。
- 函式名稱是否為 `courseHello`。
- 是否已儲存。
- 是否回到試算表畫面查看 toast 提示。

### 檢核點

- GAS 專案可開啟。
- 程式可成功執行。
- 學員有保存執行成功截圖。

## 02:20–03:00 GitHub repo 建立

### 教學目標

讓學員建立 GitHub repo，作為後續教材、程式、成果網頁與版本管理的地方。

> GitHub 對很多老師來說可能比較陌生。  
> 但我們今天不從工程師角度教 GitHub。  
> 我們把 GitHub 當作一個可以保存教材、程式、學習歷程和作品連結的地方。  
> 後面如果要做互動網頁、部署 Vercel、或保存 Python 程式，GitHub 都會是核心工具。

### 操作步驟

#### 1. 建立帳號

開啟 GitHub，登入或申請帳號。

#### 2. 建立新 repo

repo 名稱建議：

```text
earth-science-summer-2026-yourname
```

#### 3. 勾選建立 README.md

若忘記勾選，進入 repo 後再新增 README.md。

#### 4. 建立資料夾結構

可以在 GitHub 網頁上建立，也可以先用 README 表示。

建議結構：

```text
earth-science-summer-2026-yourname/
├── week01_setup/
├── week02_ground_shaking/
├── week03_eq_location/
├── week04_seismogram/
├── week05_site_effect/
├── week06_final_project/
└── README.md
```

#### 5. 編輯 README.md

貼上以下範例：

```markdown
# 北市大暑期地球科學課程

## 我的學習目標

我想學會如何把地震學與地球物理學概念，轉化成中小學可以理解與操作的教學活動。

## 每週成果

- Week 1：課程準備與工具安裝
- Week 2：簡易震動觀測活動
- Week 3：震央定位教學範例
- Week 4：波形分析小報告
- Week 5：地震防災互動網頁
- Week 6：期末教學作品
```

### 學員任務

請學員完成：

1. 建立 repo。
2. 編輯 README.md。
3. 把 repo 連結貼到課程共用表單。
4. 截圖保存到 Google Drive 的 `week01_setup`。

### 檢核點

- GitHub 帳號可登入。
- 有建立 repo。
- README.md 有內容。
- repo 連結可開啟。

# 十、第一週常見狀況與備案

## 狀況 1：學員無法安裝軟體

處理方式：

- 先完成雲端工具。
- Git、Python、Node.js 可課後補安裝。
- 教師提供截圖教學或安裝影片。
- 進階 CLI 工具不列為第一週最低通過標準。

## 狀況 2：學校帳號限制 GAS

處理方式：

- 改用個人 Google 帳號。
- 若仍無法，先看教師示範，課後補交。
- 可用截圖說明問題，不因此卡住後面進度。

## 狀況 3：GitHub 帳號驗證卡住

處理方式：

- 使用手機收信或驗證。
- 若無法完成，先用 Google Drive 暫存成果。
- 課後再完成 GitHub repo。

## 狀況 4：學員程度差異大

處理方式：

- 設定最低完成標準與進階挑戰。
- 完成快的學員可協助其他人。
- 進階學員可嘗試 Vercel 測試部署。
